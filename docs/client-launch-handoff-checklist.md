# Client Launch + Handoff Checklist

Use this when a project is approved for launch or entering final QA.

The goal is to make launch feel calm, controlled, and professional.

---

## Launch Readiness

## Client Approval

- [ ] Client approved final content
- [ ] Client approved final design/layout
- [ ] Client approved primary CTA flow
- [ ] Client confirmed contact details
- [ ] Client confirmed legal/policy content
- [ ] Client confirmed launch timing

## Content QA

- [ ] Business name is correct
- [ ] Services/offers are accurate
- [ ] Pricing/package info is accurate, if public
- [ ] Contact links work
- [ ] Booking links work
- [ ] Email addresses are correct
- [ ] Phone numbers are correct
- [ ] Location/service area is correct
- [ ] Testimonials/proof have permission to publish
- [ ] Placeholder copy is removed
- [ ] Typos/grammar checked

## Technical QA

- [ ] Mobile layout checked
- [ ] Tablet/desktop layout checked
- [ ] Navigation works
- [ ] Footer links work
- [ ] Forms submit correctly
- [ ] Confirmation/error states work
- [ ] Images load and are optimized
- [ ] Metadata/page titles checked
- [ ] Open Graph/social preview checked
- [ ] Sitemap/robots checked if applicable
- [ ] Analytics installed if applicable
- [ ] 404/not-found behavior checked
- [ ] Accessibility basics checked
  - headings
  - labels
  - keyboard focus
  - contrast
  - alt text when needed

## Domain + Deployment

- [ ] Production deploy is green
- [ ] Domain connected
- [ ] DNS records verified
- [ ] HTTPS active
- [ ] Redirects configured if needed
- [ ] Old site backup/export considered if replacing existing site
- [ ] Launch rollback path known

---

## Launch Day Steps

1. Confirm final client approval.
2. Run final production deploy.
3. Verify production URL.
4. Check homepage, key pages, and CTA path.
5. Check contact/booking/payment flows.
6. Check mobile.
7. Send launch confirmation email.
8. Record any post-launch polish tasks.
9. Offer care plan/support next step.

---

## Handoff Package

Send the client:

- Live site URL
- What was launched
- How to request edits
- Any login/admin details, if applicable
- Known post-launch items
- Recommended care plan/support option
- Reminder that small polish can happen after launch if included

Avoid sending sensitive credentials by email.

---

## Launch Confirmation Email

Subject: `Your website is live`

Hi [First Name],

Your website is live: [URL]

I’ve completed the launch checks for the core pages, mobile layout, links, and primary call-to-action path.

A few notes:

- [Note]
- [Note]
- [Note]

If you notice anything that looks off, send it my way and I’ll review it. Otherwise, the next best step is to keep the site protected and improving after launch.

If you’d like ongoing help, the Care Plan can cover updates, maintenance, small improvements, and priority support.

— Blake

---

## Post-Launch 48-Hour Check

Within 48 hours:

- [ ] Check site uptime/deployment health
- [ ] Re-check main CTA path
- [ ] Review form/booking submissions if accessible
- [ ] Review analytics install if applicable
- [ ] Fix urgent issues
- [ ] Capture nice-to-have improvements separately

---

## Post-Launch 14-Day Tune-Up

For packages that include a tune-up:

- [ ] Ask client for any launch observations
- [ ] Review analytics if available
- [ ] Fix small copy/content issues
- [ ] Fix small layout issues
- [ ] Update one or two minor items discovered after launch
- [ ] Re-offer care plan if appropriate

---

## Care Plan Transition

A client is a strong care-plan fit if:

- They want ongoing updates
- They do not want to manage site maintenance
- The site will need new content over time
- They value quick help after launch
- They want analytics/SEO/content improvements

Use care email in [`client-email-templates.md`](./client-email-templates.md).
