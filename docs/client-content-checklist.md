# Client Content + Asset Checklist

Use this after deposit/intake or immediately after kickoff.

The goal is to collect enough material to build clearly without overwhelming the client.

Recommended framing:

> You do not need to make everything perfect. Send what exists, even if it is rough. I’ll flag what is required for launch versus what can be polished later.

---

## Required for Most Projects

- [ ] Business name exactly as it should appear
- [ ] Primary contact email
- [ ] Phone number, if public
- [ ] Business location/service area, if relevant
- [ ] Current website URL, if any
- [ ] Social/profile links, if used publicly
- [ ] Primary CTA destination
  - booking link
  - contact email
  - contact form details
  - phone number
  - checkout/payment link
- [ ] Logo files
  - SVG preferred
  - PNG acceptable
- [ ] Brand colors/fonts, if available
- [ ] Service/offer descriptions
- [ ] About/founder/studio information
- [ ] Testimonials/reviews/proof
- [ ] Legal/policy details if needed

---

## Launch-Critical vs Later

Use this to avoid unnecessary delays.

### Launch-Critical

- Accurate business/contact details
- Main offer/service info
- Primary CTA
- Required legal/policy info
- Domain/DNS access if launching on client domain
- Core proof or trust signal
- Final approval from decision-maker

### Can Improve After Launch

- Extra testimonials
- Additional case studies
- New photos
- Blog posts/resources
- Advanced animations
- Secondary landing pages
- Deeper SEO content
- Minor copy polish
